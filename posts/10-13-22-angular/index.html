The basics of angular

2022-10-13
2 minute read
tech
angular

A new day! Today is Thursday, October 13th. I started the day by studying Angular for a couple hours. So, today I am going to talk a bit about Angular.
Today I learned about:  

Components

Components are apparently a big part of Angular development! You can create them in the cl or create them by hand. Obviously, creating them with the cl command is a lot easier. The command is the following:

1ng g c <COMPONENTNAME>

And you should change for the name of your component, clearly.

This command will create a new folder with 3 files inside:

  • name.component.ts
  • name.component.html
  • name.component.css

The typescript file contains the class with constructors. The html file contains the template for your component. The css file contains specific css rules for the component.

In order for Angular to recognize your component as a component you can use the @Component configuration and the following configuration options:

  • selector - It basically tells Angular to create a specific HTML tag for the component like <app-name></app-name>
  • templateUrl - It connects to our html template
  • providers - These are classes that help Angular resolve a dependency for the first time.

Data Binding

Data Binding is basically the synchronization between the model and the view. There are 3 ways of Data Binding and I am going to briefly talk about each one of them!

String Interpolation

As the name suggests, you can use a String variable directly in your HTML template.

Example ts code

1let username: string = "Beatriz";

Example HTML code

1<h1>Hello {{ username }}!</h1>

This would result in a output like:

Hello Beatriz!

Property Binding

You assign a value to a target property and that property can use your variables. So, imagine you have an api and that api has an imgPath which you can make a get request that returns an url. How would you use the url that you got from that request in your Angular application? As far as I understood it’s like this:

1<img [src] ="imgPath" alt="">

We have square brackets around the source (src) property, which represent property binding. You can also use property binding on the alt property! Imagine you make a get request and then you get imgDescription… well, it would work exactly as above!

1<img [src]="imgPath" [alt]="imgDescription">
Event Binding

We use event bindings to respond to DOM events. Like, if we click a button, the button usually does something, doesn’t it? That would be a perfect opportunity to use event binding!

1<button type="button" (click)="onClickButton()">Button</button>

When you click on the button it will activate the onClickButton() function!

That is it for today!
