A little bit about CI/CD and Jenkins

2022-10-31
4 minute read
tech
ci/cd , jenkins

I don’t really feel like making a huge introduction to this post. I will just talk a little bit about what I studied today. If you have already read Jenkins documentation you can probably skip this, I just wanted to explain it to myself.

What is CI/CD?

CI

CI stands for Continuous Integration and for that you basically merge your code into a main branch as often as possible. This way it’s easier to detect bugs and you can fix them before they get to production.

CD

CD stands for Continuous Delivery and that is you deploy your code to production as often as possible. This way you can deliver new features to your users as soon as possible. This is also known as Continuous Deployment.

How is CI/CD done?

There are a lot of tools that can help you with CI/CD. The most popular ones are Jenkins, GitLab CI, and GitHub Actions. I will be talking about Jenkins in this post.

Jenkins?

Jenkins is an open-source automation server that can help you with CI/CD. It is written in Java and it is very extensible. You can install plugins to do almost anything you want. It is also very easy to use and it has a very good documentation.

You can take a look and download Jenkins here.

How to install Jenkins?

You can download the installer here. After you download it, you can just run it and follow the steps. It’s very easy.

How to use Jenkins?

After you install Jenkins, you can access it by going to localhost:8080 in your browser. You will be asked to enter the password that was generated when you installed Jenkins. You can find it in the file C:\ProgramData\Jenkins\secrets\initialAdminPassword. After you enter the password, you will be asked to install some plugins. You can install the ones you want, but I recommend installing the suggested ones. After you install the plugins, you will be asked to create an admin user. After you create the admin user, you will be redirected to the Jenkins dashboard.

What are the main components of Jenkins?

Jobs

Jobs are the main component of Jenkins. They are basically a set of steps that you want to run. You can create jobs by going to New Item in the Jenkins dashboard. You can choose between a lot of different types of jobs. The most common ones are Freestyle Projects, Pipeline, and Multibranch Pipeline. I will be talking about Freestyle Projects and Pipeline.

Freestyle Projects

Freestyle Projects are the most basic type of job. You can create them by going to New Item in the Jenkins dashboard and choosing Freestyle Project. After you create the job, you can add steps to it. You can add steps by going to Build -> Add build step -> Execute Windows batch command. You can add as many steps as you want. You can also add parameters to the job. You can add parameters by going to Build -> Add parameter -> String Parameter. You can add as many parameters as you want. You can also add a post-build action. You can add a post-build action by going to Post-build Actions -> Add post-build action -> Archive the artifacts. You can add as many post-build actions as you want.

Pipeline

Pipeline is a more advanced type of job. You can create them by going to New Item in the Jenkins dashboard and choosing Pipeline. After you create the job, you can add a pipeline script. You can add a pipeline script by going to Pipeline -> Definition -> Pipeline script. You can add a pipeline script by writing it in the text box or by uploading a file. You can also add parameters to the job. You can add parameters by going to Pipeline -> Definition -> Pipeline script -> Parameters. You can add as many parameters as you want. You can also add a post-build action. You can add a post-build action by going to Post-build Actions -> Add post-build action -> Archive the artifacts. You can add as many post-build actions as you want.

Views

Views are basically a way to organize your jobs. You can create views by going to New View in the Jenkins dashboard. You can choose between a lot of different types of views. The most common ones are List View and My View. I will be talking about List View.

Jenkins and Docker

The most common way to use Jenkins is by using Docker. You can use Docker to run Jenkins in a container. You can take a look at the official Docker image here.

Why use Jenkins and Docker together?

The main reason would be: it’s easy. You can just run Jenkins in a container and you don’t have to install it on your machine. You can also use Docker to run your jobs in containers. You can also use Docker to run your tests in containers. You can also use Docker to run your application in containers. Jenkins has a tutorial on how to use Jenkins and Docker on a Java project here.
