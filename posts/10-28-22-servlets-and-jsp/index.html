Servlets and JSP

2022-10-28
6 minute read
tech
java , java ee , servlets

Today I decided to study about Servlets and JSP a little bit and I will talk a little bit about them.

Servlets

What are servlets?

A servlet is a Java class that extends the HttpServlet class. It is used to handle HTTP requests and responses.

When are servlets used?

Theyb are used to create dynamic web pages and applications. Dynamic web pages are web pages that are generated by the server. They are not static like HTML pages.

How to create a servlet?

To create a servlet you have to extend the HttpServlet class and override the doGet method. Here is an example:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        resp.getWriter().println("Hello World!");
5    }
6}

But that doesn’t explain a lot, does it? So let’s create a servlet that will return a HTML page. Here is the code:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        resp.setContentType("text/html");
5        resp.getWriter().println("<h1>Hello World!</h1>");
6    }
7}

The resp.setContentType("text/html"); line is used to set the content type of the response. In this case, it is HTML. The resp.getWriter().println("<h1>Hello World!</h1>"); line is used to write the HTML code to the response. But what if you want to use a variable in your HTML code? Well, you can use String Interpolation to do that. Here is an example:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        resp.setContentType("text/html");
5        String username = "Beatriz";
6        resp.getWriter().println("<h1>Hello " + username + "!</h1>");
7    }
8}

Servlet lifecycle

The servlet lifecycle is divided into 3 phases:

  • Initialization - The servlet is created and initialized.
  • Service - The servlet is called and the service method is called.
  • Destruction - The servlet is destroyed.

What is the service method?

  • doGet - It is called when the servlet is called with a GET request.
  • doPost - It is called when the servlet is called with a POST request.

But can you only make GET and POST requests? No, you can make other requests too. Here is a list of the most common requests:

  • GET - It is used to retrieve data from the server.
  • POST - It is used to send data to the server.
  • PUT - It is used to update data on the server.
  • DELETE - It is used to delete data from the server.

Servlet configuration

You can configure a servlet by using the web.xml file. Here is an example:

1<servlet>
2    <servlet-name>MyServlet</servlet-name>
3    <servlet-class>com.example.MyServlet</servlet-class>
4</servlet>
5<servlet-mapping>
6    <servlet-name>MyServlet</servlet-name>
7    <url-pattern>/my-servlet</url-pattern>
8</servlet-mapping>

The servlet-name tag is used to set the name of the servlet. The servlet-class tag is used to set the class of the servlet. The url-pattern tag is used to set the URL pattern of the servlet. The URL pattern is used to map the servlet to a URL. In this case, the servlet will be mapped to the /my-servlet URL. So if you want to access you servlet in localhost you have to go to http://localhost:8080/my-servlet.

Servlet context

The servlet context is used to store data that can be accessed by all servlets. Here is an example:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        ServletContext context = getServletContext();
5        context.setAttribute("username", "Beatriz");
6    }
7}

The context.setAttribute("username", "Beatriz"); line is used to set the username attribute to Beatriz. To get the value of the username attribute you have to do this:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        ServletContext context = getServletContext();
5        String username = (String) context.getAttribute("username");
6    }
7}

What are containers?

Containers are used to run servlets, the most common container is the Tomcat container. You can download it here.

JSP

What is JSP?

JSP (Java Server Pages) is a script language. You use it to create dynamic pages with HTML.

How to create a JSP?

To create a JSP you have to create a .jsp file. Here is an example:

<%@ page contentType="text/html" pageEncoding="UTF-8" %>
<html>
    <head>
        <title>Hello World!</title>
    </head>
    <body>
        <h1>Hello World!</h1>
    </body>
</html>

How to use JSP?

To use JSP you have to create a servlet that extends the HttpServlet class and override the doGet method. Here is an example:

1public class MyServlet extends HttpServlet {
2    @Override
3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
4        RequestDispatcher dispatcher = req.getRequestDispatcher("/WEB-INF/my-jsp.jsp");
5        dispatcher.forward(req, resp);
6    }
7}

The RequestDispatcher dispatcher = req.getRequestDispatcher("/WEB-INF/my-jsp.jsp"); line is used to get the RequestDispatcher object. The dispatcher.forward(req, resp); line is used to forward the request to the JSP. The WEB-INF folder is used to store JSP files. You can’t access JSP files that are not in the WEB-INF folder.

JSP syntax

Here is a list of the most common JSP syntax:

  • <% %> - It is used to write Java code.
  • <%= %> - It is used to write Java code and print the result.
  • <%@ %> - It is used to write JSP directives.
  • <%! %> - It is used to write JSP declarations.
  • <jsp:include page=""/> - It is used to include a JSP file.
  • <jsp:forward page=""/> - It is used to forward a request to a JSP file.

JSP declarations

JSP declarations are used to declare variables and methods. Here is an example:

<%@ page contentType="text/html" pageEncoding="UTF-8" %>
<%! String username = "Beatriz"; %>
<html>
    <head>
        <title>Hello World!</title>
    </head>
    <body>
        <h1>Hello <%= username %>!</h1>
    </body>
</html>

Practical example

Let’s create a servlet that will take data from the database and a JSP file that will display the data. Here is the servlet:

 1public class MyServlet extends HttpServlet {
 2    @Override
 3    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
 4        try {
 5            Class.forName("com.mysql.jdbc.Driver");
 6            Connection connection = DriverManager.getConnection("jdbc:mysql://localhost:3306/my_database", "root", "");
 7            Statement statement = connection.createStatement();
 8            ResultSet resultSet = statement.executeQuery("SELECT * FROM users");
 9            List<User> users = new ArrayList<>();
10            while (resultSet.next()) {
11                User user = new User();
12                user.setId(resultSet.getInt("id"));
13                user.setUsername(resultSet.getString("username"));
14                user.setPassword(resultSet.getString("password"));
15                users.add(user);
16            }
17            req.setAttribute("users", users);
18            RequestDispatcher dispatcher = req.getRequestDispatcher("/WEB-INF/my-jsp.jsp");
19            dispatcher.forward(req, resp);
20        } catch (ClassNotFoundException | SQLException e) {
21            e.printStackTrace();
22        }
23    }
24}

Here is the JSP file:

<%@ page contentType="text/html" pageEncoding="UTF-8" %>
<html>
    <head>
        <title>Users</title>
    </head>
    <body>
        <table>
            <tr>
                <th>ID</th>
                <th>Username</th>
                <th>Password</th>
            </tr>
            <c:forEach items="${users}" var="user">
                <tr>
                    <td>${user.id}</td>
                    <td>${user.username}</td>
                    <td>${user.password}</td>
                </tr>
            </c:forEach>
        </table>
    </body>
</html>

Connecting a mysql database to a servlet

To connect a mysql database to a servlet you have to add the mysql connector to the project. Here is an example:

1<dependency>
2    <groupId>mysql</groupId>
3    <artifactId>mysql-connector-java</artifactId>
4    <version>8.0.19</version>
5</dependency>

How to use the mysql connector?

To use the mysql connector you have to create a servlet that extends the HttpServlet class and override the doGet method. Here is an example:
